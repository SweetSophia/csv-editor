# REVIEW FINDINGS

## 🔴 CRITICAL - Must Fix

### 1. File Permission Too Permissive
**File:** `bindings.go:252`
```go
if err := os.WriteFile(path, data, 0644); err != nil {
```
**Issue:** `0644` allows group and others to read the file. CSV files may contain sensitive data.
**Fix:** Use `0600` (owner read/write only).

### 2. Regex DoS (ReDoS) in Find
**File:** `find.ts:18-29`
```javascript
function buildRegex(query: string, opts: FindOptions): RegExp | null {
    // ...
    return new RegExp(escaped, flags);
}
```
**Issue:** No protection against catastrophic backtracking. Evil regex like `([a-z]+)+b` on long strings causes CPU hang.
**Fix:** Add regex timeout guard or use `DOM` `RegExp` with try/catch + abort for long-running patterns.

### 3. Memory Explosion - Full Row Copies in Undo
**File:** `state.ts:97-117`
```javascript
function pushHistory(state, rowsAfter, headerAfter): EditableState {
    return {
        ...state,
        rows: rowsAfter,
        // rowsBefore and rowsAfter store ENTIRE row arrays
        history: [...state.history, { rowsBefore: state.rows, rowsAfter, ... }],
    };
}
```
**Issue:** Each undo entry stores complete row copies. A 100k-row × 50-col file = ~10MB per undo entry. 50 undo states = 500MB.
**Fix:** Use structural diffs or ring buffer with compression for very large datasets.

## 🟠 HIGH - Should Fix

### 4. Paste Overflow Edge Case
**File:** `App.tsx:556-571`
```javascript
const overflowsRows = b.r0 + clipRows > rows.length;
const wouldExtend = overflowsRows || overflowsCols;
```
**Issue:** If user pastes at a position that extends beyond table, it extends. But `b.r0` is clamped to `lastRow` in selection, so it can't actually overflow via normal selection. However, if paste logic ever allows unbounded position, this would corrupt data.
**Fix:** Ensure bounds checking is consistent throughout paste flow.

### 5. Auto-fit Column Measurement DOM Pollution
**File:** `App.tsx:789-848`
```javascript
const span = document.createElement('span');
// ... styles ...
document.body.appendChild(span);
// ... measure ...
span.remove();
```
**Issue:** On huge tables (100k+ rows), iterates with stride but still creates/removes DOM element per column. If user auto-fits many columns rapidly, could cause jank.
**Fix:** Batch measurement requests with `requestAnimationFrame` or use `ResizeObserver`.

### 6. matchesByCell Recalculation on Every Match Change
**File:** `VirtualTable.tsx:162-177`
```javascript
const matchesByCell = useMemo(() => {
    const map = new Map<string, CellMatchInfo>();
    for (let i = 0; i < matches.length; i++) { ... }
    return map;
}, [matches, currentMatchIndex]);
```
**Issue:** The entire map rebuilds when ANY match changes. With thousands of matches, this creates GC pressure.
**Fix:** Use incremental Map updates or index by cell address separately.

### 7. No File Size Limit
**File:** `bindings.go:155-159`
```go
func (b *Bindings) LoadFile(path, ...) {
    data, err := os.ReadFile(path)
    // ...
}
```
**Issue:** No check for file size. A 2GB CSV would crash the app.
**Fix:** Add max file size check (e.g., 500MB) with user-friendly error.

### 8. Child Process Error Not Handled
**File:** `bindings.go:394-396`
```go
if err := cmd.Start(); err != nil {
    wailsRuntime.EventsEmit(b.ctx, "file:error", "new window: "+err.Error())
}
// cmd.Start() errors are emitted but execution continues silently
```
**Issue:** Only `cmd.Start()` error is caught. `cmd.Wait()` is never called, so we don't know if child actually started or crashed immediately.
**Fix:** Use `cmd.Run()` or track process and handle `Wait()`.

## 🟡 MEDIUM - Consider Fixing

### 9. MOVE_COLS Index Calculation Subtlety
**File:** `state.ts:314-325`
```javascript
const order: number[] = [];
if (direction === 'left') {
    for (let i = 0; i < startIndex - 1; i++) order.push(i);  // BUG: should be i < startIndex
    for (let i = startIndex; i < end; i++) order.push(i);
    order.push(startIndex - 1);
```
**Issue:** When moving columns left, the iteration `i < startIndex - 1` misses column `startIndex - 1`. Should be `i < startIndex` to include it, then skip it in the moving block.
**Verification needed:** This appears to be off-by-one. Trace through: moving cols 2-3 left should reorder to [0, 1, 2, 3] → [0, 2, 3, 1].

### 10. Encoding Detection Limited
**File:** `encoding.go:48-57`
```go
func Detect(data []byte) Encoding {
    if bytes.HasPrefix(data, utf8BOM) { return UTF8BOM }
    if utf8.Valid(data) { return UTF8 }
    return CP932  // Always CP932 if not UTF-8
}
```
**Issue:** Always falls back to CP932 for non-UTF-8. Files in ISO-8859-1, EUC-JP, etc. will be mis-decoded.
**Fix:** Add detection for other common encodings or show encoding selection dialog.

### 11. No Loading Indicator
**File:** `App.tsx`
**Issue:** Large file loads block the UI thread (Go parses CSV before returning). No spinner or progress indication.
**Fix:** Stream parsing or web workers for large files, or at minimum show loading state.

### 12. Unsaved Changes on Close - No Prompt
**File:** `main.go:167-170`
```go
fileMenu.AddText("Close Window", keys.CmdOrCtrl("w"), func(_ *menu.CallbackData) {
    wailsRuntime.Quit(b.ctx)
})
```
**Issue:** Closing window via menu doesn't check for unsaved changes. Only `onBeforeClose` in bindings saves state, doesn't prompt.
**Fix:** Add unsaved changes check in close handler.

### 13. No Undo Limit
**File:** `state.ts`
**Issue:** History grows unbounded. Each edit adds a full row copy.
**Fix:** Cap history at reasonable limit (e.g., 100 entries) with oldest-first eviction.

### 14. Selection Range Display Bug
**File:** `VirtualTable.tsx:617`
```javascript
selection.focus.columnIndex === Math.max(0, maxColumns - 1)
```
**Issue:** Checking if selection "spans full row" by comparing to `maxColumns - 1`. But `maxColumns` includes the row number column. Should be `maxColumns` (since data cols are 0-indexed and max is already maxColumns-1 for last data col).
**Actually correct:** `maxColumns` is the count, so `maxColumns - 1` is the last data column index. This is correct.

## 🟢 LOW - Nice to Have

```javascript
const NUMERIC_RE = /^-?(?:\d+\.?\d*|\.\d+)(?:[eE][+-]?\d+)?$/;
```
**Issue:** Doesn't handle locale-specific number formats (1.234,56 in European locales).
**Fix:** Consider locale-aware parsing or explicit format selection.


**Issue:** Cmd+Z/Cmd+Shift+Z are captured by table but not when find bar input is focused.

```javascript
{contextMenu && (
```
